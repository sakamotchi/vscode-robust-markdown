# テスト手順書 - Markdownプレビュー基盤

## 概要

Markdownプレビュー基盤（Phase 1）のテスト手順を記載します。
手動テストは F5 で起動した Extension Development Host 上での操作確認が基本です。

## 前提条件

- VSCodeで本リポジトリを開いている
- `npm run compile` または `npm run watch` でビルドが完了している
- F5 を押して Extension Development Host が起動できる状態

---

## 手動テスト

### ケース 1: アイコンボタンからプレビューを起動する

**手順:**

1. F5 を押して Extension Development Host を起動する
2. 任意の `.md` ファイルを開く
3. エディタータブ右端に `$(preview)` アイコンが表示されることを確認する
4. アイコンをクリックする

**期待結果:**

- Markdownのプレビューが新しいVSCodeタブ（WebviewPanel）で開く
- パネルタイトルがファイル名になっている
- 見出し・リスト・コードブロックが正しくレンダリングされている

**確認結果:**

- [x] OK / NG

---

### ケース 2: 右クリックメニューからプレビューを起動する

**手順:**

1. F5 を押して Extension Development Host を起動する
2. 任意の `.md` ファイルを開く
3. エディター上で右クリックする
4. コンテキストメニューに「Open Robust Markdown Preview」が表示されることを確認する
5. メニューを選択する

**期待結果:**

- Markdownのプレビューが新しいVSCodeタブで開く

**確認結果:**

- [x] OK / NG

---

### ケース 3: 非Markdownファイルではアイコンが表示されない

**手順:**

1. F5 を押して Extension Development Host を起動する
2. `.ts` または `.json` ファイルを開く
3. エディタータブ右端を確認する

**期待結果:**

- `$(preview)` アイコンが表示されない

**確認結果:**

- [x] OK / NG

---

### ケース 4: 重複パネルが生成されない

**手順:**

1. F5 を押して Extension Development Host を起動する
2. 任意の `.md` ファイルを開く
3. アイコンをクリックしてプレビューを開く
4. プレビューパネルの隣に元のMarkdownエディタを表示した状態にする
5. 再度アイコンをクリックする

**期待結果:**

- 新しいパネルは生成されない
- 既存のプレビューパネルにフォーカスが移る

**確認結果:**

- [x] OK / NG

---

### ケース 5: 複数ファイルのプレビューを同時に開く

**手順:**

1. F5 を押して Extension Development Host を起動する
2. `README.md` を開いてプレビューを起動する
3. 別の `.md` ファイルを開いてプレビューを起動する

**期待結果:**

- それぞれ独立したVSCodeタブ（WebviewPanel）でプレビューが表示される
- 両方のタブが同時に存在する

**確認結果:**

- [x] OK / NG

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| 空のMarkdownファイル | エラーなく空のプレビューを表示 | |
| 不正なMarkdown構文（例: 閉じていないコードブロック） | エラーメッセージをインライン表示、パネルは維持 | |
| ファイルを閉じた後にプレビューパネルを閉じる | エラーなく正常に閉じる | |

## 回帰テスト

Phase 2以降の開発完了時にPhase 1の動作が維持されていることを確認します。

- [ ] アイコン・右クリックの両方からプレビューが起動できる
- [ ] 重複パネルが生成されない
- [ ] 複数ファイルのプレビューを同時に開ける
