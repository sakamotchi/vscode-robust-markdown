# テスト手順書 - 自動更新・AI更新耐性

## 概要

自動更新・AI更新耐性（Phase 2）のテスト手順を記載します。
手動テストは F5 で起動した Extension Development Host 上での操作確認が基本です。

## 前提条件

- VSCodeで本リポジトリを開いている
- `npm run compile` または `npm run watch` でビルドが完了している
- F5 を押して Extension Development Host が起動できる状態

---

## 手動テスト

### ケース 1: 編集後にプレビューが自動更新される

**手順:**

1. F5 を押して Extension Development Host を起動する
2. 任意の `.md` ファイルを開き、プレビューを起動する
3. エディターでテキストを追加・変更する（保存しない）
4. 約1000ms待つ

**期待結果:**

- プレビューが最新の編集内容に更新される

**確認結果:**

- [x] OK / NG

---

### ケース 2: 保存時にプレビューが自動更新される

**手順:**

1. F5 を押して Extension Development Host を起動する
2. 任意の `.md` ファイルを開き、プレビューを起動する
3. エディターでテキストを変更する
4. ⌘S（保存）する

**期待結果:**

- プレビューが保存後すぐに更新される

**確認結果:**

- [x] OK / NG

---

### ケース 3: 高頻度編集でもクラッシュしない（AI更新耐性）

**手順:**

1. F5 を押して Extension Development Host を起動する
2. 任意の `.md` ファイルを開き、プレビューを起動する
3. エディターで素早くキーを連打して短時間に多数の編集イベントを発生させる（例：数秒間ランダムな文字を入力し続ける）
4. 1000ms待つ

**期待結果:**

- プレビューパネルがクラッシュ・閉じることなく維持される
- 最後の編集から1000ms後にプレビューが更新される

**確認結果:**

- [x] OK / NG

---

### ケース 4: パースエラー時もパネルが維持される

**手順:**

1. F5 を押して Extension Development Host を起動する
2. 任意の `.md` ファイルを開き、プレビューを起動する
3. ファイルの内容を一時的に大量の特殊文字や不完全な構文に変更する（例：`<><><><>` を大量に入力）
4. 1000ms待つ

**期待結果:**

- プレビューパネルが閉じない
- エラーが発生した場合はエラーメッセージがインライン表示される（または特殊文字がそのまま表示される）
- 内容を正常なMarkdownに戻すと通常表示に戻る

**確認結果:**

- [x] OK / NG

---

### ケース 5: プレビューのないファイルの編集は無視される

**手順:**

1. F5 を押して Extension Development Host を起動する
2. `file_a.md` を開いてプレビューを起動する
3. `file_b.md` を開く（プレビューは起動しない）
4. `file_b.md` を編集する

**期待結果:**

- `file_a.md` のプレビューが更新されない（`file_b.md` の編集が無視される）
- エラーも発生しない

**確認結果:**

- [x] OK / NG

## エッジケース

| ケース | 期待動作 | 確認結果 |
|--------|---------|---------|
| プレビューを閉じた直後に編集イベントが発生 | 存在しないパネルへの更新が無視される | |
| 複数ファイルのプレビューを同時に開いて編集 | それぞれのパネルが独立して更新される | |
| デバウンス中にパネルを閉じる | タイマーがクリアされ、エラーが発生しない | |

## 回帰テスト

Phase 1 の動作が維持されていることを確認します。

- [ ] アイコン・右クリックの両方からプレビューが起動できる
- [ ] 重複パネルが生成されない
- [ ] 複数ファイルのプレビューを同時に開ける
