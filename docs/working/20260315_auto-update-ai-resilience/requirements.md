# 要件定義書 - 自動更新・AI更新耐性

## 概要

Markdownプレビューを編集・保存に連動して自動更新する機能を実装する。
AIエージェント（Claude Codeなど）が高頻度でファイルを書き換えても、プレビューがクラッシュしない堅牢性を実現するため、デバウンス処理とエラーハンドリングをあわせて実装する。

## 背景・目的

Phase 1ではプレビューの初回表示のみを実装した。本フェーズでは、以下の2点を追加する：

1. **自動更新**: ファイル編集・保存のたびにプレビューを最新状態に保つ
2. **AI更新耐性**: 1秒間に複数回の更新イベントが発生しても、デバウンス（1000ms）により安定動作を維持する

## 要件一覧

### 機能要件

#### F-101: リアルタイム自動更新

- **説明**: ファイル編集中（未保存）の変更でプレビューを自動更新する
- **受け入れ条件**:
  - [ ] ファイルを編集して1000ms待つとプレビューが更新される
  - [ ] プレビューが開いていないファイルの編集イベントは無視される

#### F-102: 保存時自動更新

- **説明**: ファイル保存時にプレビューを自動更新する
- **受け入れ条件**:
  - [ ] ファイルを保存するとプレビューが更新される
  - [ ] プレビューが開いていないファイルの保存イベントは無視される

#### F-103: デバウンス処理

- **説明**: 更新イベントを1000msデバウンスして高頻度更新に対応する
- **受け入れ条件**:
  - [ ] 短時間に複数の編集イベントが発生しても、最後のイベントから1000ms後に1回だけ更新される
  - [ ] パネルごとに独立したデバウンスタイマーが動作する（ファイルAの更新がファイルBのタイマーに影響しない）

#### F-104: パースエラー耐性

- **説明**: Markdownパースエラーが発生してもプレビューパネルを維持し、エラーをインライン表示する
- **受け入れ条件**:
  - [ ] 不正なMarkdown（意図的に例外を発生させた場合）でもパネルが閉じない
  - [ ] エラーメッセージがプレビュー内にインライン表示される
  - [ ] 次の更新で正常なMarkdownが来れば通常表示に戻る

### 非機能要件

- **AI更新耐性**: AIエージェントの高頻度書き込みでもクラッシュしない
- **リソース管理**: パネル破棄時に対応するデバウンスタイマーも確実に解放する
- **独立性**: 複数ファイルのデバウンスタイマーが互いに干渉しない

## スコープ

### 対象

- `src/extension.ts` — ドキュメントイベントリスナーの登録
- `src/previewManager.ts` — `updatePreview()` 実装、デバウンスタイマー管理
- `src/debounce.ts` — デバウンスユーティリティの新規作成

### 対象外

- Mermaid対応 → Phase 3
- UI的なローディングインジケーター（スコープ外）

## 実装対象ファイル（予定）

- `src/debounce.ts` — 新規作成
- `src/previewManager.ts` — `updatePreview()` 追加、デバウンスタイマーMap追加
- `src/extension.ts` — `onDidChangeTextDocument` / `onDidSaveTextDocument` リスナー追加

## 依存関係

- Phase 1実装済みの `PreviewManager`（`openPreview` / `renderToPanel`）
- VSCode API: `vscode.workspace.onDidChangeTextDocument`、`vscode.workspace.onDidSaveTextDocument`

## 既知の制約

- デバウンス待機時間は **1000ms** 固定（設定値ではない）
- `onDidChangeTextDocument` はすべての編集イベントを発火するため、`panels` マップの存在チェックによるフィルタリングが必須
