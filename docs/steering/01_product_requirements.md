## 1. プロダクト概要

### 1.1 プロダクト名
AI Safe Markdown Preview（VSCode拡張機能）

### 1.2 目的
AIコーディングエージェント（Claude Codeなど）がMarkdownファイルを頻繁に更新する環境でも、クラッシュしないMarkdownプレビューを提供するVSCode拡張機能を開発する。

### 1.3 ビジョン
「AIエージェントによる高頻度更新にも耐えられる、安定・軽量なMarkdownプレビュー環境」

### 1.4 主要な価値提案
- **AI更新耐性**: デバウンス処理とエラー隔離により、頻繁な更新でもクラッシュしない
- **シームレスな起動**: エディタータブのアイコンまたは右クリックから即座に起動
- **Mermaid対応**: コードブロック内の図表をそのまま表示
- **VSCode標準UI**: ネイティブタブで複数ファイルを自然に扱える

## 3. 主要機能

### 3.1 機能一覧

#### Phase 1: Markdownプレビュー基盤
| 機能ID | 機能名 | 説明 | 状態 |
|--------|--------|------|------|
| F001 | Markdownレンダリング | `marked` を使って Markdown → HTML に変換し WebviewPanel で表示 | 完了 |
| F002 | プレビュー起動（アイコン） | エディタータブ右端のアイコンボタンからプレビューを起動 | 完了 |
| F003 | プレビュー起動（右クリック） | エディター上での右クリックメニューからプレビューを起動 | 完了 |
| F004 | 重複パネル防止 | 同一ファイルのプレビューパネルを重複生成しない | 完了 |

#### Phase 2: 自動更新・AI更新耐性
| 機能ID | 機能名 | 説明 | 状態 |
|--------|--------|------|------|
| F101 | リアルタイム自動更新 | 編集中のドキュメント変更でプレビューを自動更新 | 未着手 |
| F102 | 保存時自動更新 | ファイル保存時にプレビューを自動更新 | 未着手 |
| F103 | デバウンス処理 | 更新イベントを1000msデバウンスして高頻度更新に対応 | 未着手 |
| F104 | パースエラー耐性 | Markdownパースエラーでもパネルを維持し、エラーをインライン表示 | 未着手 |

#### Phase 3: Mermaid対応
| 機能ID | 機能名 | 説明 | 状態 |
|--------|--------|------|------|
| F201 | Mermaid図表表示 | ` ```mermaid ``` ` ブロックを図として表示 | 未着手 |
| F202 | Mermaidエラー隔離 | 特定の図のエラーを他の図・本文に波及させない | 未着手 |

## 4. 非機能要件

### 4.1 互換性要件
| 要件ID | 要件 | 対応バージョン |
|--------|------|--------------|
| NFR001 | VSCode | 1.110.0以上 |
| NFR002 | Node.js | 18以上 |

### 4.2 性能要件
| 要件ID | 要件 | 目標値 |
|--------|------|--------|
| NFR101 | デバウンス待機時間 | 1000ms |
| NFR102 | メモリ使用量 | 最小限（不要なWebviewPanelは即時破棄） |

### 4.3 可用性要件
| 要件ID | 要件 | 説明 |
|--------|------|------|
| NFR201 | オフライン動作 | mermaid.jsをバンドルするためインターネット接続不要 |
| NFR202 | クラッシュ耐性 | パースエラー・Mermaidエラーでもプレビューパネルを維持 |

---

## 5. 制約条件

- VSCode拡張機能フレームワークの制限に従う
- Webview の CSP（Content Security Policy）の制約を遵守する
- mermaid.js はバンドル（オフライン対応優先、CDN不使用）
